# サイト改善監査メモ

## 確認日
2026-05-01

## 確認対象
- `index.html`
- `style.css`
- `README.md`
- `articles/` 配下の自動生成記事

## 見つかった主な問題

### 1. 記事一覧の見出しが弱い
トップページの記事カードが `seo`、`python`、`a8` のような短い語だけになっており、読者が読む理由を判断しにくい状態でした。

### 2. 自動生成記事の内容が似すぎている
複数の記事で「結論・理由・手順・注意点」がほぼ同じ構成・文面になっており、検索意図への深さが不足しています。今後は各記事ごとに具体例、失敗例、チェックリスト、内部リンクの役割を変える必要があります。

### 3. README が空に近い
リポジトリの目的、公開方法、編集対象ファイル、運営方針が書かれていませんでした。保守しやすさと引き継ぎ性の面で弱いです。

### 4. CSS が1行に圧縮されていて保守しにくい
`style.css` は機能していますが、1行にまとまっているため修正差分が読みにくく、今後の改善でミスが起きやすいです。

### 5. アフィリエイト導線の説明がやや薄い
PR表記はありますが、読者の悩みから広告への流れを記事ごとにもっと自然に分ける余地があります。

## 実施した改善

- README を整備
- トップページの記事一覧を読者向けタイトル・説明文に変更
- 監査メモを追加

## 次に改善すべきこと

1. `style.css` を整形して保守しやすくする
2. 各記事に固有の内容を追加する
3. `sitemap.xml` と `robots.txt` を追加する
4. 全ページに canonical URL を追加する
5. 記事下部の関連リンクを、記事テーマごとに最適化する
6. 広告リンクのクリック前説明を具体化する

## 許可・確認が必要なこと

- 公開URL、独自ドメインの有無
- Google Search Console の登録状況
- A8.net の提携案件・掲載ルールの確認
- アクセス解析タグを入れてよいか
